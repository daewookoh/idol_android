# WebViewScreen

> 외부 웹페이지를 앱 내에서 표시하는 범용 화면

---

## 🎯 주요 기능

- URL 기반 웹페이지 로드
- 선택적 앱바 표시 (타이틀 있을 경우)
- 로딩 인디케이터 표시
- 백버튼 처리

* 없음 (stateless 화면)

* 없음 (외부 URL만 로드)

## 🔄 화면 플로우

### 1️⃣ 진입
* Navigation: 다양한 화면에서 URL과 타이틀을 파라미터로 전달
* 파라미터:
  * `url`: 로드할 웹페이지 URL (필수)
  * `title`: 앱바에 표시할 타이틀 (옵션, 없으면 앱바 미표시)

### 2️⃣ 사용자 액션

| 액션 | 설명 | 결과 |
|------|------|------|
| 백버튼 클릭 | 하드웨어 백버튼 또는 앱바 뒤로가기 | 이전 화면으로 이동 |
| 웹페이지 조작 | 링크 클릭, 스크롤, 입력 | 웹페이지 내에서 정상 동작 |

### 3️⃣ 종료
* 백버튼 클릭 시 → 이전 화면으로 이동

## 🎨 UI 구성

| 요소 | 스펙 |
|------|------|
| **앱바** | 타이틀이 있을 경우에만 표시 |
| **WebView** | 전체 화면 (앱바 아래 영역) |
| **로딩 인디케이터** | 중앙 배치, 메인 색상 |

### WebView 설정
* JavaScript 활성화
* DomStorage 활성화
* 줌 지원 (컨트롤은 숨김)
* 와이드 뷰포트 사용
* 오버뷰 모드로 로드

---

## 🔒 보안 설정

* 앱 내에서 웹페이지 로드 (외부 브라우저로 이동하지 않음)
* JavaScript 활성화 (웹페이지 정상 동작 보장)

---

## 📝 주의사항

* 🌐 **범용 화면**: 다양한 목적으로 재사용 가능 (공지사항, 이벤트, 외부 링크 등)
* 📱 **타이틀 옵션**: 타이틀을 전달하지 않으면 전체 화면 WebView로 표시
* 🔗 **외부 링크**: 모든 링크는 앱 내에서 처리되며 외부 브라우저로 이동하지 않음
* ⚡ **로딩 표시**: 페이지 로드 중 로딩 인디케이터 자동 표시
